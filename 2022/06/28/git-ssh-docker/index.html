git&ssh&docker

发表于2022-06-28|更新于2022-07-19|Linux开发
|阅读量:

这里只记录个人最常用的命令，其余现用现查

git

git 部分由于现在大多是个人开发所以rebase 和 pull request 都没有记录

提交与查看

  • git add <filename> / git add .
    • git stash clear 清空暂存空间
    • git reset <filename> 将已经add但没commit的文件移除
  • git commit -m <message>
  • git status / git log
    • git status 红色表明有新的变更

回退

  • git reset <commitID> 进行回退，commitID为使用git log查看的哈希值
    • git reset <commitID> --hard 不保存所有变更

分支

  • git check out -b <name> <template> 以template为模板建立以name为名的新分支
    • template 选填如为空则以当前所在分支为模板
  • git branch 查看所有分支
    • git checkout <BrachName> 切换分支

合并

  • git merge <BranchName> 合并分支的变更

仓库

  • git remote add origin <link>
    • git remote -v 查看仓库连接状态
  • git clone
  • git push origin <Local-BranchName>
    • 原命令 git push origin <Local-BranchName>:<Remote-BranchName>
    • git push -f origin master 强制推送
    • git push origin --delete <Remote-BranchName> 删除远程主机的master的该分支
    • 如果首次使用的是 git push -u origin <Local-BranchName>，那么以后可直接使用 git push
  • git push --set-upstream origin <Local-BranchName>
    • 在远程仓库新建分支，解决当前所在分支与仓库所在远端上流分支不匹配问题
  • git pull origin <Remote-BranchName>:<Local-BranchName>
    • 同时完成 fetch 和 merge 的操作
  • git pull --rebase origin <Remote-BranchName>:<Local-BranchName>
    • 合并远程和本机分支，解决大部分push失败问题

提交规范(type 部分)

  • sync：同步主线或分支的bug
  • merge：代码合并
  • revert：回滚到上一个版本
  • chore：构建过程或辅助工具的变动
  • test：增加测试
  • perf：优化相关，比如提升性能、体验
  • refactor：重构（既不是新增功能，也不是修改bug的代码变动）
  • style：格式（不影响代码运行的变动）
  • docs：文档
  • fix / to：修复bug
  • feat：新功能

ssh

登陆与配置

  • ssh-keygen 创建rsa密钥

  • 在 .ssh/config 下配置服务器：

  • ssh-copy-id <ServerName> 免密登录

    • 然后直接使用 ssh myserver 即可直接登录

  • ssh <User>@<HostName> -p <PortNumber>

    • 使用用户名User登录端口号为PortNumber的主机名为HostName的服务器
    • 实例： ssh acs@152.135.154.181 -p 20000

传文件

  • 本地 ==> 服务器

    • 传文件
    • 传文件夹 ：使用 -r 递归复制

  • 服务器 ==> 本地

    • 传文件
    • 传文件夹

docker

官网：Post-installation steps for Linux | Docker Documentation

目前正在学习阶段，部分命令之后会补充 (这行话没了的时候就是补充完毕的时候)

镜像

  • docker pull xxx 拉取某镜像
  • docker rmi xxx 删除某镜像
  • docker save -o <Name>.tar <ImageName> 将这个镜像导出到本地Name.tar文件中
  • docker load -i <Name>.tar 将这个镜像文件加载出来

容器

  • docker ps 列出容器
    • docker ps -a 显示所有包括没运行的
    • docker ps -s 显示总文件大小
  • docker run -itd <ContainerName> 创建并启动容器
    • docker create -it docker run -itd <ContainerName> 创建容器
  • docker start <ContainerName> 启动容器
  • docker stop <ContainerName> 停止容器
  • docker restart <ContainerName> 重启容器
  • docker rm <ContainerName> 删除容器
  • docker attach <ContainerName> 进入容器
    • exit 直接退出
    • ctrl+p + ctrl+q 挂起后退出
  • docker export -o <Name>.tar <ContainerName> 将容器导出到本地
  • docker import <Name>.tar <ContainerName> 将文件导入成镜像并命名为 ContainerName
